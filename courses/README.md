# 课程体系

本目录包含 AI 时代第一性原理计算机科学课程体系的六大模块。

## 课程模块

1. [Module A: 基础](./module-a-foundations.md) - 计算模型、抽象、复杂度
2. [Module B: 语言与执行](./module-b-language-execution.md) - 解释器、编译器、运行时
3. [Module C: 单机系统](./module-c-single-node-systems.md) - 操作系统、文件系统、内存管理
4. [Module D: 数据系统](./module-d-data-systems.md) - 数据库、存储引擎、索引
5. [Module E: 网络与分布式](./module-e-network-distributed.md) - 协议、一致性、容错
6. [Module F: AI 与搜索系统](./module-f-ai-search-systems.md) - 搜索引擎、推荐系统、ML 框架

## 模块与系统仓库的对应关系

| 模块 | 对应系统仓库 |
|------|-------------|
| Module A | 无独立仓库，为其他模块提供理论基础 |
| Module B | tiny-interpreter, simple-compiler |
| Module C | mini-os, simple-fs |
| Module D | storage-engine, tx-manager |
| Module E | consensus, dist-kv |
| Module F | 待规划 |

## 学习路径建议

### 路径 1: 自底向上（推荐）

适合想要系统性理解计算机科学的学习者。

```
Module A (基础)
  ↓
Module B (语言与执行)
  ↓
Module C (单机系统)
  ↓
Module D (数据系统)
  ↓
Module E (网络与分布式)
  ↓
Module F (AI 与搜索)
```

### 路径 2: 兴趣驱动

适合有特定目标的学习者，可以跳过部分模块。

- **想理解数据库**：Module A → Module D
- **想理解分布式系统**：Module A → Module E
- **想理解编译器**：Module A → Module B
- **想理解操作系统**：Module A → Module C

### 路径 3: 项目驱动

适合想要快速上手的学习者，边做边学。

1. 选择一个感兴趣的系统仓库
2. 阅读对应模块的文档
3. 按需补充前置知识
4. 开始构建系统

## 每个模块的结构

每个模块文档包含以下部分：

1. **模块目的与范围** - 这个模块要解决什么问题
2. **必学概念与不变量清单** - 核心概念与必须理解的不变量
3. **标志性产物** - 要构建的系统
4. **验收标准** - 如何判断是否真正理解
5. **推荐项目顺序** - 如果有多个项目，应该按什么顺序做
6. **参考资料** - 推荐的论文、书籍、博客

## 学习建议

### 建议 1: 理解优先于实现

不要急于写代码，先理解：
- 这个系统要解决什么问题？
- 核心抽象是什么？
- 关键不变量是什么？
- 可能的失败场景有哪些？

### 建议 2: 验证优先于优化

先实现正确性，再考虑性能：
- 先写测试，明确不变量
- 再写实现，让测试通过
- 最后优化，保持测试通过

### 建议 3: 简化优先于完整

不要追求功能完整，而要追求概念清晰：
- 删除非必要特性
- 保留核心抽象
- 用最简单的方式实现

### 建议 4: 记录优先于记忆

不要依赖记忆，而要依赖文档：
- 记录设计决策
- 记录不变量
- 记录失败场景
- 记录实验结果

## 与 AI 协作

在学习过程中，建议遵循 [AI 协作协议](../principles/ai-collaboration-protocol.md)：

1. **规格先行** - 先明确要做什么，再让 AI 生成代码
2. **生成实现** - 让 AI 生成实现与测试
3. **约束测试** - 验证不变量与失败场景
4. **重构压复杂度** - 简化实现，保持测试通过
