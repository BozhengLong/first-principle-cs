# Principles 原则文档

本目录包含我们构建和验证系统时遵循的四大核心原则。

## 四大原则

1. [接口与不变量](./interface-and-invariants.md) - 明确系统边界，定义必须始终成立的性质
2. [失败模型](./failure-model.md) - 识别所有可能的失败场景，设计容错机制
3. [验证优先](./verification-first.md) - 测试先行，用属性测试和故障注入验证不变量
4. [AI 协作协议](./ai-collaboration-protocol.md) - 与 AI 协作的固定流程

## 原则之间的关系

这四个原则形成一个完整的闭环：

```
接口与不变量 → 定义"什么必须成立"
      ↓
失败模型 → 定义"什么可能出错"
      ↓
验证优先 → 测试"不变量是否真的成立"
      ↓
AI 协作协议 → 用 AI 加速"规格-实现-验证"循环
      ↓
   (回到接口与不变量，开始下一轮重构)
```

## 使用建议

在开始任何新系统之前，建议按以下顺序应用这些原则：

1. 先读 [接口与不变量](./interface-and-invariants.md)，明确系统的边界与核心性质
2. 再读 [失败模型](./failure-model.md)，识别可能的失败场景
3. 然后读 [验证优先](./verification-first.md)，设计测试策略
4. 最后读 [AI 协作协议](./ai-collaboration-protocol.md)，了解如何与 AI 高效协作

## 为什么需要这些原则

在 AI 时代，我们可以快速生成大量代码，但这也带来了新的风险：

- **风险 1**：代码能运行，但我们不知道它保证了什么
- **风险 2**：测试能通过，但我们不知道它测试了什么
- **风险 3**：系统能工作，但我们不知道它在什么情况下会失败

这些原则帮助我们在快速迭代的同时，保持对系统的深度理解。
